# LinkZentrum

**LinkZentrum** — это система автоматического размещения ссылок (линкопомойка) для SEO-продвижения. Система позволяет управлять донорами ссылок, загружать ссылки для размещения и автоматически вставлять их на страницы доноров с возможностью клоакинга.
Система доведена до MVP, пользоваться можно, но внутри очень много недоделано из задуманного.

> **Version:** 0.1.9a  
> **PHP:** 5.4.1+  
> **License:** Proprietary

## ✨ Возможности

- **Управление донорами** — добавление, удаление и проверка работоспособности доноров
- **Загрузка ссылок** — массовая загрузка ссылок из файла с поддержкой ключевых слов
- **Клоакинг** — показ ссылок только поисковым ботам (по User-Agent, IP или комбинации)
- **Ротация ссылок** — автоматическое распределение ссылок между страницами
- **Разбавление анкоров** — возможность смешивания анкорных списков с дополнительными словами
- **Статистика** — подсчёт размещённых ссылок, доноров и страниц
- **Многопоточная проверка** — проверка доноров на работоспособность (18 потоков)
- **Кэширование** — XML-кэширование ссылок для ускорения отдачи
- **Адаптивный интерфейс** — веб-интерфейс на базе Gumby CSS Framework

## 🚀 Установка

### 1. Подготовка

1. Скопируйте файлы проекта в корневую директорию вашего домена
2. Убедитесь, что домен соответствует значению `_DOM_` в `conf.php`

### 2. Настройка прав доступа

Установите права `0777` на следующие директории:

```bash
chmod 0777 XML/
chmod 0777 data/
chmod 0777 data/cache/
```

### 3. Конфигурация

Отредактируйте файл `conf.php`:

```php
define('_DOM_', 'yourdomain.com');     // Ваш домен
define('DB_HOST', 'localhost');         // Хост БД
define('DB_NAME', 'Linkzentrum');       // Имя БД
define('DB_USER', 'root');              // Пользователь БД
define('DB_PASS', 'your_password');     // Пароль БД
```

### 4. Создание базы данных

1. Создайте базу данных с именем, указанным в `conf.php`
2. Откройте главную страницу в браузере
3. Система автоматически проверит требования и предложит установку
4. Нажмите кнопку **Install** для создания таблиц

### 5. Первый вход

Пароль по умолчанию: **`A123456a`**

> ⚠️ **Важно:** Сразу после первого входа смените пароль в разделе **Настройки**

## 📖 Использование

### Веб-интерфейс

#### Главная страница (`/`)
- Общая статистика (ссылки, доноры, страницы)
- Кнопка проверки доноров
- Индикатор установки

#### Доноры (`/?donor`)
- Управление списком доноров
- Привязка доноров к категориям ссылок
- Проверка работоспособности (кнопка "Проверка доноров")

#### Ссылки (`/?links`)
- Загрузка ссылок из файла (UTF-8)
- Поддерживаемый формат:
  ```html
  <a href="http://domain.tld/page/key.html">keyword</a>
  ```
- Просмотр ссылок по доменам
- Удаление ссылок (по домену или все)
- Выгрузка ссылок

#### Настройки (`/?options`)
- **Ссылок на страницу** — количество ссылок на одной странице
- **Показывать ссылки** — режим клоакинга:
  - `0` — Всем
  - `1` — По UA
  - `2` — По IP
  - `3` — По UA+IP
- **Разбавленных анкоров** — процент разбавления (0-100)
- **Обновлять через** — интервал обновления ссылок (часы)
- **Ссылок без ключа** — процент ссылок без ключевых слов
- **Распределение ссылок**:
  - `1 стр. = 1 реципиент` — одна страница = один получатель
  - `1 стр. = N реципиентов` — одна страница = несколько получателей
- **Размещать ссылки** — где размещать:
  - `Везде` — на всех страницах
  - `Только на главной` — только на главной странице
- **Файл с добавочными словами** — файл для разбавления анкоров
- **Смена пароля** — установка нового пароля администратора

### Проверка доноров

Для проверки доноров на работоспособность:

1. Перейдите в раздел **Доноры**
2. Нажмите **Проверка доноров**
3. Система отправит запросы на все домены с User-Agent `LINKZENTRUM`
4. Ищет метку `_WORD_` в ответе
5. Обновляет статус донора (1 = работает, 0 = не работает)

Проверка выполняется через `checker.php` с использованием RollingCurl (18 потоков).

## 🔌 API

### get.php

Основной обработчик запросов ботов.

**Параметры:**

| Параметр | Описание | Пример |
|----------|----------|--------|
| `useragent` | User-Agent запроса | `Googlebot` |
| `pageurl` | URL страницы | `http://example.com/page` |
| `ch` | Кодировка | `utf-8` или `win1251` |
| `ip` | IP адрес | `66.249.66.1` |

**Пример запроса:**
```
http://lzentrum.my/get.php?useragent=Googlebot&pageurl=http://example.com/&ch=utf-8&ip=66.249.66.1
```

**Ответ:**
```html
<ul>
<li><a href="http://link1.com">Anchor 1</a></li>
<li><a href="http://link2.com">Anchor 2</a></li>
</ul><!--Render in 0.0234 sec -->
```

### check.php

Проверка доноров (вызывается через `?check=donors`).

**Режимы:**
- `?check=donors` — проверка доноров
- `?check=delete` — удаление нерабочих доноров

### Таблицы

| Поле | Тип | Описание |

#### `urls`
Ссылки для размещения

| Поле | Тип | Описание |
|------|-----|----------|
| `id` | INT | ID записи (AUTO_INCREMENT) |
| `url` | VARCHAR(150) | URL ссылки |
| `keyword` | VARCHAR(150) | Анкор (ключевое слово) |
| `posted_cn` | INT | Счётчик размещений |
| `domain` | INT(11) | ID домена (из urls_domains) |

#### `urls_domains`
Домена ссылок (реципиенты)

| Поле | Тип | Описание |
|------|-----|----------|
| `id` | INT | ID записи (AUTO_INCREMENT) |
| `domain` | VARCHAR(75) | Домен |
| `data_add` | INT(11) | Дата добавления (timestamp) |
| `status` | TINYINT(1) | Статус (1 = активен, 0 = не активен) |

#### `counters`
Счётчики статистики

| Поле | Тип | Описание |
|------|-----|----------|
| `id` | TINYINT(1) | ID (всегда 1) |
| `counter_links` | INT(11) | Всего ссылок размещено |
| `counter_pages` | INT(11) | Всего страниц обработано |

## 🎛️ Настройки

### Формат файла настроек (`.options`)

Настройки хранятся в сериализованном формате PHP:

```php
a:9:{
  s:5:"count";s:1:"5";           // Ссылок на страницу
  s:7:"ch_time";i:18000;         // Время обновления (сек)
  s:9:"anch_perc";d:7;           // Ссылок без ключа (%)
  s:8:"anch_mix";d:50;           // Разбавление анкоров (%)
  s:5:"cloak";s:1:"0";           // Режим клоакинга
  s:11:"post_regime";s:1:"1";    // Режим размещения
  s:9:"where_put";s:1:"0";       // Где размещать
  s:9:"mixw_file";s:15:"...";    // Файл слов
  s:7:"def_cat";i:1;             // Категория по умолчанию
}
```

### Файлы для разбавления анкоров

Расположение: `data/words/`

Формат: текстовый файл в кодировке UTF-8, одно слово на строку.

Пример:
```
лучший
качественный
профессиональный
надёжный
```

## 🔒 Безопасность

### Авторизация

- Пароли хранятся в хэшированном виде (SHA1 + соль)
- Файл с хэшем: `data/.pwd`
- Сессии PHP для авторизации

### Защита файлов

- `.htaccess` в корне ограничивает доступ
- `.htaccess` в `XML/` запрещает прямой доступ
- `robots.txt` запрещает индексацию

### Рекомендации

1. **Смените пароль по умолчанию** сразу после установки
2. **Ограничьте доступ** к админ-панели по IP
3. **Используйте HTTPS** для защиты сессии
4. **Регулярно обновляйте** списки ботов в `USER-AGENTS.txt` и `IP-RANGES.txt`
5. **Делайте бэкапы** базы данных регулярно

## 🔧 Troubleshooting

### Ошибки при установке

**Проблема:** Не создаются таблицы БД

**Решение:**
- Проверьте права доступа пользователя БД
- Убедитесь, что БД существует
- Проверьте лог ошибок MySQL

**Проблема:** Ошибка "Domain: lzentrum.my"

**Решение:**
- Убедитесь, что `_DOM_` в `conf.php` совпадает с `$_SERVER["SERVER_NAME"]`

### Ошибки при работе

**Проблема:** Не отображаются ссылки

**Решение:**
- Проверьте статус донора в таблице `donors_domains`
- Убедитесь, что файл кэша создан в `XML/{hash}/`
- Проверьте настройки клоакинга

**Проблема:** Медленная работа

**Решение:**
- Увеличьте `innodb_buffer_pool_size` в MySQL
- Настройте кэширование (memcached/redis)
- Уменьшите частоту обновления ссылок

**Проблема:** Ошибка памяти

**Решение:**
- Увеличьте `memory_limit` в PHP
- Уменьшите `max_input_vars`
- Оптимизируйте размер списков ссылок

### Логи

Система использует функцию `derlog()` для логирования (в данный момент пустая).

Для включения логирования отредактируйте `inc/functions.php`:

```php
function derlog($error){
    file_put_contents(_DATA_ . 'error.log', date('Y-m-d H:i:s') . ' - ' . $error . PHP_EOL, FILE_APPEND);
}
```
